# Willkommen in deinem Second Brain!

Schön, dass du hier bist. Du hast gerade den ersten Schritt gemacht, um dein Wissen dauerhaft zu strukturieren, zu verknüpfen und wirklich nutzbar zu machen. Dieses System wächst mit dir — fang einfach an, der Rest ergibt sich.

---

## Onboarding in 3 Schritten

### Schritt 1: Obsidian Plugins installieren

Öffne **Einstellungen → Community Plugins → Durchsuchen** und installiere:

| Plugin | Status | Warum? |
|--------|--------|--------|
| **Dataview** | Pflicht | Automatische Listen und Tabellen in MOCs |
| **Templater** | Optional | Erweiterte Template-Variablen (Datum, Titel) |
| **Calendar** | Optional | Kalenderansicht für Daily Notes |
| **Kanban** | Optional | Kanban-Boards für Projektmanagement |

> Nach der Installation: Aktiviere die Plugins unter **Einstellungen → Community Plugins**.

### Schritt 2: Dashboard erkunden

Öffne [[Home]] — das ist dein zentrales Dashboard. Von dort erreichst du alle Bereiche deines Second Brain:
- Aktive Projekte und deren Status
- Lebensbereiche (Areas)
- Wissensressourcen
- Maps of Content (MOCs) zur Navigation

### Schritt 3: Erste Notiz erstellen

1. Gehe in den Ordner `00 - Inbox`
2. Erstelle eine neue Notiz
3. Wähle ein Template aus `05 - Templates/` (z.B. `Daily Note.md`)
4. Schreibe deine ersten Gedanken auf

---

## Dein erster Tag

Hier sind 5 konkrete Aktionen, mit denen du sofort loslegen kannst:

1. **Tages-Notiz anlegen** — Erstelle eine Notiz in `00 - Inbox/Daily Notes` mit dem `Daily Note.md` Template. Schreibe auf, was dich heute beschäftigt.

2. **Ein Projekt anlegen** — Erstelle in `01 - Projects` ein aktives Projekt mit dem `Project.md` Template. Was arbeitest du gerade? Ein Nebenprojekt, ein Lernziel, eine Idee?

3. **Eine Ressource eintragen** — Liest du gerade ein Buch, einen Artikel oder schaust einen Kurs? Trage es in `03 - Resources` mit dem `Resource.md` Template ein.

4. **Workflow Guide lesen** — Öffne den [[Workflow Guide]] und verstehe, wie die Ordnerstruktur, Tags und Wiki-Links zusammenspielen.

5. **Graph View öffnen** — Drücke `Ctrl+G` (Mac: `Cmd+G`) und sieh dir an, wie deine Notizen als Netzwerk verbunden sind. Je mehr Notizen du erstellst und verlinkst, desto wertvoller wird diese Ansicht.

## Mit Claude Code starten

Dieses Vault ist für die Arbeit mit Claude Code optimiert. Starte einfach `claude` im Vault-Ordner — er liest die [[CLAUDE]] Konfiguration automatisch.

Hier sind 5 Befehle, mit denen du sofort loslegen kannst:

```
"Erstelle eine Daily Note für heute"
```

```
"Lege ein neues Projekt an: Spanisch lernen"
```

```
"Fasse alle meine Ressourcen zum Thema Produktivität zusammen"
```

```
"Welche offenen Tasks habe ich in meinen Projekten?"
```

```
"Verlinke meine Notiz über Zettelkasten mit passenden Ressourcen"
```

Claude kennt alle Templates, Konventionen und die Ordnerstruktur. Du musst ihm nichts erklären — er weiß, wo Notizen hingehören und wie sie verknüpft werden.
