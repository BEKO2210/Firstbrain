### PROA-09 Failure Detail

`getOutdatedReferences` in `maintain-utils.cjs` uses `link.resolvedPath` (lines 384, 386, 388) but the link-map.json schema produced by the v1.0 scanner uses `link.targetPath`. The field `resolvedPath` does not exist in any link-map entry. The function silently returns an empty array regardless of vault state.

**Fix:** Replace `link.resolvedPath` with `link.targetPath` on lines 384, 386, 388.

### PROA-08 Partial Detail

`getStaleProjects()` in `maintain-utils.cjs` is missing an `isTemplate` guard. The `05 - Templates/Project.md` template (which has `type: project`, `status: active`) is incorrectly reported as a stale project. The `isExcluded()` helper checks `isTemplate` but is only used by `runConsistencyChecks`, not `getStaleProjects`. By comparison, `briefing-utils.cjs getStaleProjects` correctly guards with a `notePath.startsWith('01 - Projects/')` check.

**Fix:** Add `if (noteData.isTemplate) continue;` at the top of the loop in `getStaleProjects`.

### Additional Integration Gaps (Non-Blocking)

**triage getTargetFolder('review'):** `create-utils.cjs TEMPLATE_MAP` uses `'weekly'` and `'monthly'` as keys but `'review'` is the actual frontmatter type. `getTargetFolder('review')` returns null, preventing folder-move suggestion for review-type notes in inbox. Edge case — review notes don't normally land in inbox root.

**triage SKILL.md type count:** Claims "12 valid types" on line 65 but lists 11. Documentation-only, no functional impact.
