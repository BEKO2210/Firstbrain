# Entscheidung: KI-Geschaeftsmodell

## Kontext

> Ich will mit KI-Automatisierung ein zweites Standbein aufbauen. Es gibt verschiedene Modelle — ich muss mich fuer eines entscheiden, um fokussiert loszulegen statt alles gleichzeitig zu probieren.

## Optionen

### Option A: KI-Chatbot-Entwicklung
- Pro: Hohe Nachfrage, viele Tutorials, schneller Einstieg
- Contra: Niedriger Preis (Commoditisierung), viel Konkurrenz, wenig Recurring Revenue

### Option B: KI-Workflow-Automatisierung (Agenten + n8n)
- Pro: Hoehere Wertschoepfung, Wartungsvertraege moeglich, weniger Konkurrenz, nutzt meinen Developer-Hintergrund
- Contra: Laengerer Sales-Cycle, komplexer zu erklaeren, braucht technisches Setup

### Option C: KI-Beratung / Coaching
- Pro: Kein Code noetig, sofort startbar, hohe Stundensaetze
- Contra: Nicht skalierbar, tauscht Zeit gegen Geld, keine Hebelwirkung

## Entscheidung

> **Gewaehlt: Option B — KI-Workflow-Automatisierung**

## Begruendung

- Mein Developer-Hintergrund ist ein klarer Wettbewerbsvorteil — die meisten AAA-Anbieter koennen nur No-Code
- Wartungsvertraege schaffen Recurring Revenue (→ [[Zettel — Die 1000-Euro-Regel fuer Side Business]])
- n8n + Claude API als Stack ist DSGVO-konform (self-hosted)
- Passt zu meiner verfuegbaren Zeit: Workflows einmal bauen, dann laeuft es
- Ich kann mein eigenes Familienprojekt als Portfolio-Stueck nutzen (→ [[KI-Workflow fuer Familienorganisation]])

## Konsequenzen

- Portfolio mit 3 Beispiel-Workflows aufbauen
- n8n + Claude API Stack vertiefen
- Fokus auf KMUs im DACH-Raum (deutschsprachig, DSGVO wichtig)
- LinkedIn-Praesenz auf "KI-Automatisierung fuer KMUs" ausrichten
- Chatbot-Projekte bewusst ablehnen (Fokus!)
