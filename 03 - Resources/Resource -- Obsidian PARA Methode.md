# Resource -- Obsidian PARA Methode

## Summary

> PARA ist ein universelles Organisationssystem für digitale Informationen. Es sortiert alles in vier Kategorien: Projects (zeitgebunden), Areas (fortlaufend), Resources (Referenzmaterial), Archive (abgeschlossen). Die Stärke liegt in der Einfachheit und der Handlungsorientierung.

## Key Ideas

1. **Handlungsorientierung**: Projects haben ein klares Ende, Areas nicht. Diese Unterscheidung verhindert, dass offene Aufgaben zu ewigen "Bereichen" werden.
2. **Durchlässigkeit**: Informationen wandern zwischen Kategorien: ein Projekt wird zum Archiv, eine Resource wird zum Projekt-Input.
3. **Minimale Ordnerstruktur**: 4 Top-Level-Ordner reichen. Alles darunter entsteht organisch.

## Notes

Firstbrain erweitert PARA um:
- `00 - Inbox/` als Eingangszone (nicht in Standard-PARA)
- `05 - Templates/` für standardisierte Notiz-Erstellung
- `06 - Atlas/MOCs/` als Navigationsebene (Maps of Content)
- `07 - Extras/` für Anhänge und Medien

Die Erweiterung funktioniert gut, weil die Kernprinzipien (handlungsorientiert, durchlässig) erhalten bleiben.

## Application

- Firstbrain Vault nutzt PARA als Grundstruktur
- Jede neue Note wird automatisch in die richtige Kategorie sortiert (via `/create`)
- `/triage` klassifiziert Inbox-Notizen nach PARA-Kategorien
- Archivierung erfolgt manuell oder via Projekt-Abschluss
